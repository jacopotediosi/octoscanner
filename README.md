# Octoscanner

A static analysis tool for OctoPrint plugins. A static analysis tool for OctoPrint plugins. Detects usage of features deprecated or removed between OctoPrint versions, security issues, and common bad practices.

> [!WARNING]
> Please note that this tool is still a very early version and false positives are possible, as well as false negatives (the tool does not yet detect all deprecations/removals, especially not frontend related things).

## Features

- **Automatic rule generation** - Analyzes multiple OctoPrint versions to generate [Semgrep](https://semgrep.dev) rules for deprecations and removals
- **Hand-written rules** - Includes additional rules for security issues and code quality problems commonly found in plugins
- **Built-in downloader** - Fetches sources of multiple OctoPrint versions and plugins with a single command
- **Integrated scanner** - Scans one or more plugins at once with clear, actionable output

## The Generator

The core of Octoscanner is the `generator` module, which analyzes OctoPrint sources and automatically produces Semgrep rules.

The generator runs a two-phase pipeline:

1. **Analyzers** (`generator/analyzers/`) - Parse OctoPrint source code to extract useful information (e.g., class hierarchies and deprecation markers).

2. **Processors** (`generator/processors/`) - Consume analyzer results to build Semgrep rules (e.g., deprecation and removal rules).

## Usage

Below are the most common usages. For a complete list of commands and options, see `octoscanner --help`.


### Download plugins
```bash
# Download Telegram plugin to plugins_src/telegram
octoscanner download plugins telegram
```

### Scan a plugin

```bash
# Scan a plugin
octoscanner scan /path/to/my_plugin

# Scan multiple plugins
octoscanner scan /path/to/plugin1 /path/to/plugin2

# Filter by rule category or specific rule file
octoscanner scan /path/to/my_plugin -r deprecation -r removal/python_removal.yaml
```

#### Ignore false positives

Add [markers](https://semgrep.dev/docs/ignoring-files-folders-code#ignore-code-through-nosemgrep) (e.g., `# nosemgrep: REM-0001`, `// nosemgrep`) in your plugin code to skip specific lines or blocks.

You can also exclude specific rules during scanning with `--exclude-rule` (e.g., `octoscanner scan /path/to/my_plugin --exclude-rule REM-0001 --exclude-rule DEP-0002`).

#### CI Integration

Octoscanner returns exit code `1` if any scanned plugin has issues, making CI integration straightforward. Use `octoscanner scan` in your workflow, optionally with `--no-color` or `--format json`.

Alternatively, [integrate Semgrep directly into your CI](https://semgrep.dev/docs/semgrep-ci/sample-ci-configs) using Octoscanner's generated rules.

Consider [Opengrep](https://www.opengrep.dev/) as an alternative - in my tests it cuts scan time in half compared to Semgrep, saving CI minutes.

### Generate rules

Rules are typically generated by the Octoscanner maintainer for manual review and refinement. If you just want to scan plugins, use the included pre-generated rules.

Instead, if you're feeling adventurous...

```bash
# Step 1. Download OctoPrint sources for the most relevant versions
octoscanner download octoprint all

# Step 2. Dry-run: analyze all downloaded versions, print what would be generated
octoscanner generate all

# Step 3. Generate and save rules to disk
octoscanner generate all --save
```

You can also analyze only the diff between two OctoPrint versions and generate rules for those:

```bash
octoscanner generate 1.10.0 1.11.0
```
